Eenvoudige overerving

Doel: een klasse overerven van een andere klasse.

Theorie

Met overerving kan een klasse de eigenschappen van een andere klasse overnemen. De klasse die overerft heet de subklasse, de klasse waarvan overgeërfd wordt de basisklasse. 

// basisklasse Dier
public class Dier
{
    public string Naam { get; set; }
    public int Leeftijd { get; set; }
}

// Hond erft Naam en Leeftijd over van Dier
public class Hond : Dier
{
   // eigen properties van Hond
   public string Ras { get; set; }       
   public bool IsGetraind { get; set; }
}

Voorbeeld van een Hond instantie:

Hond jacky = new Hond {
   Naam = "Jacky",        // van Dier
   Leeftijd = 3,          // van Dier
   Ras = "Schnauzer",     // eigen
   IsGetraind = true      // eigen
};

Abstracte klasse

Doel: klassen maken die moeten overgeërfd worden om te kunnen gebruiken.

Theorie

Een abstracte klasse is een klasse die je niet rechtstreeks kan instantiëren. Ze dient enkel als basisklasse voor subklassen. Je markeert ze met het keyword abstract. 

// Kan niet rechtstreeks aangemaakt worden: new Vorm() → fout!
public abstract class Vorm
{
   public string Kleur { get; set; }
}

Je moet ze overerven om te kunnen gebruiken:

// Erf over om te gebruiken
public class Cirkel : Vorm
{
   public double Straal { get; set; }
}
public class Rechthoek : Vorm
{
   public double Breedte { get; set; }
   public double Hoogte { get; set; }
}

Gebruik een abstracte klasse wanneer het concept te algemeen is om rechtstreeks te gebruiken (Vorm, Dier, …)

Cirkel c1 = new Cirkel { Kleur = "groen", Straal = 13.7 }; // ok
Rechthoek r1 = new Rechthoek { Kleur = "blauw", Breedte = 210, Hoogte = 133 }; // ok
Vorm v1 = new Vorm { Kleur = "rood" }; // fout! Vorm mag niet rechtstreeks gebruikt worden

Abstracte property of methode

Doel: methodes maken die moeten geïmplementeerd worden in overgeërfde klassen.

Theorie

Een abstracte methode of property heeft geen implementatie in de basisklasse. Elke subklasse is verplicht om ze te implementeren met override. 

public abstract class Vorm
{
   public string Kleur { get; set; }

   // abstracte property: subklasse MOET deze implementeren
   public abstract double Oppervlakte { get; }

   // abstracte methode: subklasse MOET deze implementeren
   public abstract void Herschaal(double factor);
}

public class Cirkel : Vorm
{
   public double Straal { get; set; }
   public override double Oppervlakte { get  { return Math.PI * Straal * Straal; } }
   public override void Herschaal(double factor)
   {
      Straal *= factor;
   }
}

public class Rechthoek : Vorm
{
   public double Breedte { get; set; }
   public double Hoogte { get; set; }
   public override double Oppervlakte { get  { return Breedte * Hoogte; } }
   public override void Herschaal(double factor)
   {
      Breedte *= factor;
      Hoogte *= factor;
   }
}

Merk op hoe automatisch de juiste versie van Oppervlakte en Herschaal() geselecteerd wordt:

List<Vorm> vormen = new List<Vorm>();
vormen.Add(new Cirkel { Kleur = "groen", Straal = 13.7 });
vormen.Add(new Rechthoek { Kleur = "blauw", Breedte = 210, Hoogte = 133 });

foreach (Vorm v in vormen)
{
   Console.WriteLine($"oppervlakte: {v.Oppervlakte}"); // compiler kiest automatisch de juiste versie van Oppervlakte
   v.Herschaal(2); // compiler kiest automatisch de juiste versie van Herschaal()
   Console.WriteLine($"na herschaling x2: {v.Oppervlakte}");
}

Dit noemt men polymorfisme ("meerdere vormen"): één methode voor de buitenwereld, intern verschillende implementaties—een uiterst belangrijk begrip in OO!

Gebruik abstract wanneer je subklassen wil verplichten een methode te implementeren.

is en as

Doel: het type van een object controleren met is en casten naar een subklasse met as.

Theorie

Met is controleer je of een object van een bepaald type is. Met as cast je naar een subklasse — het resultaat is null als het type niet klopt (geen exception). Codevoorbeeld: 

// basisklasse
class Gebruiker {
   public int Id { get; set; }
   public string Naam { get; set; }
}

// afgeleide klassen
class Medewerker : Gebruiker {
   public int ToegangsCode { get; set; }
}
class Klant : Gebruiker {
   public string KlantNummer { get; set; }
}

// maak enkele gebruikers
List<Gebruiker> gebruikers = new List<Gebruiker>();
gebruikers.Add(new Medewerker() { Id = 1, Naam = "An Janssen", ToegangsCode = 4872 });
gebruikers.Add(new Klant() { Id = 2, Naam = "Marc Pieters", KlantNummer = "KL-00123" });
gebruikers.Add(new Medewerker() { Id = 3, Naam = "Sara Leclercq", ToegangsCode = 9341 });
gebruikers.Add(new Klant() { Id = 4, Naam = "Jonas De Smet", KlantNummer = "KL-00456" });

// itereer over gebruikers, en parse telkens met "is" en "as"
foreach (Gebruiker user in gebruikers) {
   if (user is Medewerker) {
      Medewerker mw = user as Medewerker;
      Console.WriteLine($"Medewerker {mw.Naam} heeft toegangscode {mw.ToegangsCode}");
   }
   if (g is Klant) {
      Klant kl = user as Klant;
      Console.WriteLine($"Klant {kl.Naam} heeft klantnummer {kl.KlantNummer}");
   }
}

In moderne C# kan je is en as in één stap combineren:

// "is" en "as" in één stap gecombineerd
if (user is Medewerker as mw) {
   Console.WriteLine($"Medewerker {mw.Naam} heeft toegangscode {mw.ToegangsCode}");
}

base

Doel: de constructor en methodes van de basisklasse aanroepen vanuit een subklasse.

Theorie

Met base roep je iets op uit de basisklasse. Er zijn twee gebruiken:

  • base(...) in de constructor — roept de constructor van de basisklasse op
  • base.Methode() in een override — roept de implementatie van de basisklasse op
public class Fiets
{
   public string Merk { get; set; }
   public int Versnellingen { get; set; }

   public Fiets(string merk, int versnellingen)
   {
      Merk = merk;
      Versnellingen = versnellingen;
   }

   public override string ToString() => $"{Merk}, {Versnellingen} versnellingen";
}

public class ElektrischeFiets : Fiets
{
   public int Kwh { get; set; }

   // base(...): roept constructor van Fiets op
   public ElektrischeFiets(string merk, int versnellingen, int kwh) : base(merk, versnellingen)
   {
      Kwh = kwh;
   }

   // base.ToString(): bouwt verder op de output van Fiets
   public override string ToString() => $"{base.ToString()}, batterij {Kwh} kWh";
}
